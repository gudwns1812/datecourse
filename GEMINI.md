# GEMINI.md - 프로젝트 지침

이 파일은 이 프로젝트(datecourse)를 진행할 때 준수해야 할 핵심 원칙과 가이드라인을 담고 있습니다.

## 1. 프로젝트 구조

## 1. 프로젝트 구조 및 모듈별 지침

작업 범위에 따라 아래의 상세 지침 파일을 반드시 먼저 읽고 준수하십시오.

- **Backend 작업 시**: `backend/BACKEND.md` 참조 (Java 21, Spring Boot, 계층형 아키텍처, TDD)
- **Frontend 작업 시**: `frontend/FRONTEND.md` 참조 (Next.js, 컴포넌트 설계)

## 2. 개발 원칙 및 공통 로직

- **모든 문서는 한글로 작성**: 코드 내 주석, 커밋 메시지, 문서화 작업은 모두 한글을 우선으로 합니다.
- **모듈 격리**: 백엔드와 프론트엔드는 독립된 모듈로 관리하며, 직접적인 파일 참조를 금지합니다.
- **계획 우선**: 작업시 plans폴더에 md 파일을 만들고 승인을 받으십시오.
- **기술 스택**:
    - Backend: Java 21, Spring Boot 3.4.3, JPA, MySQL, Lombok
    - Frontend: Next.js

## 3. 커밋 가이드라인

1. 머릿말(Type): feat:, fix:, chore: 등과 같은 유형을 먼저 명시해야 합니다.
2. 모듈명: 그 뒤에 [backend] 또는 [frontend]와 같이 대괄호를 사용하여 모듈을 구분합니다.
3. 내용: 최종적으로 한글로 변경 목적과 내용을 작성합니다.

- 예: `feat: [backend] XX기능 추가`

## 4. 기타 지시사항

- **테스트 주도 개발 (TDD)**: 기능을 추가하거나 수정, 리팩토링할 때는 **실제 구현 코드를 작성하기 전에 반드시 테스트 코드를 먼저 작성**해야 합니다.
- 프로젝트 루트의 `plan.md`를 참조하여 구조 변경 이력을 확인하고, 새로운 대규모 작업 시 계획을 먼저 업데이트합니다.
