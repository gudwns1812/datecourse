# GEMINI.md - 프로젝트 지침

이 파일은 이 프로젝트(datecourse)를 진행할 때 준수해야 할 핵심 원칙과 가이드라인을 담고 있습니다.

## 1. 프로젝트 구조

이 프로젝트는 Gradle 기반의 멀티 모듈 프로젝트입니다.

- `backend/`: Spring Boot 기반의 백엔드 모듈 (Java 25)
- `frontend/`: 프론트엔드 모듈 (향후 React, Next.js 등 도입 예정)
- 루트 디렉토리: Gradle Wrapper 및 멀티 모듈 설정 파일 (`settings.gradle`, `build.gradle`)

## 2. 개발 원칙

- **모든 문서는 한글로 작성**: 코드 내 주석, 커밋 메시지, 문서화 작업은 모두 한글을 우선으로 합니다.
- **모듈 분리 준수**: 백엔드와 프론트엔드 코드는 각자의 디렉토리 내에서 독립적으로 관리하며, 상호 의존성이 생길 경우 루트의 Gradle 설정을 통해 명확히 관리합니다.
- **기술 스택**:
    - Backend: Java 21, Spring Boot 4.0.1, JPA, MySQL, Lombok
    - Frontend: Next.js

## 3. 커밋 가이드라인

- 커밋 메시지는 한글로 작성합니다.
- 변경 사항의 목적(왜)과 내용(무엇을)을 명확히 기술합니다.
- 예: `feat: [backend] XX기능 추가`

## 4. 기타 지시사항

- 새로운 기능을 추가하거나 버그를 수정할 때는 반드시 관련 테스트 코드를 함께 작성하거나 업데이트해야 합니다.
- 프로젝트 루트의 `plan.md`를 참조하여 구조 변경 이력을 확인하고, 새로운 대규모 작업 시 계획을 먼저 업데이트합니다.
